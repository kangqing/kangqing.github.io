##Linux版本:阿里云镜像仓库的CentOS7
CentOS-7-x86_64-DVD-1908.iso
###安装配置jdk
首先检查是否安装过jdk

rpm -qa | grep jdk

如果有则删除,XXX为上面命令查到的结果，按个删除

yum remove XXX

然后去Oracle官网下载jdk,我下载的是1.8的 jdk-8u221-linux-x64.rpm
赋予权限：

chmod 777 jdk-8u221-linux-x64.rpm

解释：777代表用户、用户组、其他都是7的权限，7代表什么呢？7代表的是读、写、执行三种权限

  • 读4

  • 写2

  • 执行1
    加起来正好是7
    接下来安装jdk的命令是：

    rpm -ivh jdk-8u221-linux-x64.rpm

  • 注意：jdk会被默认安装到 /usr/java 的路径下

  • 编辑/etc/profile文件

    vi /etc/profile

    按Insert或者I键，切换成编辑模式。
    找到内容末尾，输入下面一段话。
    在unset i的前面输入

    export JAVA_HOME=/usr/java/jdk1.8.0_221-amd64
export CLASSPATH=.:$JAVA_HOME/jre/lib/rt.jar:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar
export PATH=$PATH:$JAVA_HOME/bin

    ###（！！！注意：JAVA_HOME的路径是你实际安装的JDK的路径或者tar解压包的路径我这里是/usr/java/jdk1.8.0_221-amd64，千万别写错了）

  • ###然后按Esc 输入:wq保存退出

生效环境变量
对于/etc/profile编写完成之后是不够的，还需要最后一个步骤，就是让刚刚我们修改的文件变成有效起来，所以我们再输入一个命令，让修改生效。
生效命令：

source /etc/profile

验证安装是否成功：

java -version

#安装Tomcat
1 安装tomcat之前必须安装配置好jdk，因为Tomcat启动需要jdk支持
2 下载Tomcat 我是在官网下载的Tomcat9的tar.gz压缩包上传到linux的
3 解压缩

tar -zxvf apache-tomcat-9.0.24.tar.gz

cd 进入解压后的目录pwd获取当前目录路径得到/root/installFile/apache-tomcat-9.0.24

4 配置环境变量

vi /etc/profile

按i进入编辑模式
接着jdk配置后面加上下面这句话，注意=后面是刚才pwd得到的tomcat解压缩得到的路径

export CATALINA_HOME=/root/installFile/apache-tomcat-9.0.24

5 按Esc 输入:wq 保存退出
6 使配置生效

source /etc/profile

7 配置UTF-8字符集
进入tomcat下的conf文件下编辑server.xml文件

找到这块加上

URIEncoding="UTF-8"

:wq保存退出
8 进tomcat的bin目录下执行./start.sh启动tomcat

./start.sh

Tomcat启动成功
之后在windows浏览器输入ip:8080就可以得到如下页面了

  • 注意如果访问不到很大可能是防火墙没开放8080端口
    ###注意：：：centos7的防火墙改成了firewall不再是iptables
    执行如下操作：
    firewall-cmd --state  //看防火墙的状态
systemctl start firewalld.service  //如果是关闭状态则启动防火墙，否则跳过此步骤
firewall-cmd --zone=public --add-port=8080/tcp --permanent   //开启8080端口
systemctl restart firewalld.service //重启防火墙
firewall-cmd --reload //重新载入防火墙配置
firewall-cmd --list-all //查看防火墙开启的端口列表
    上面开启8080端口的解释
  • -zone=public：表示作用域为公共的；
  • -add-port=8080/tcp：添加tcp协议的端口8080；
  • -permanent：永久生效，如果没有此参数，则只能维持当前服务生命周期内，重新启动后失效；

9 关闭tomcat执行./shutdown.sh

./shutdown.sh

#安装maven
1 确保安装了jdk
2 到apache maven官网下载maven我这里最新的是3.6.2版本

3 使用xftp上传到linux
4 解压缩

tar -zxvf apache-maven-3.6.2-bin.tar.gz

5 配置环境变量

vi /etc/profile

按i进入编辑模式
加入下面一段话

export MAVEN_HOME=/root/installFile/apache-maven-3.6.2

然后把maven_home加到path路径后面:$MAVEN_HOME/bin

//注意把这句话放到最后面不然配置不成功，可能是因为顺序读取没读到Maven_home的定义呢
export PATH=$PATH:$JAVA_HOME/bin:$MAVEN_HOME/bin

:wq保存退出
使配置生效

source /etc/profile

测试是否配置成功

mvn -version

在~/.bashrc里面加一句source /etc/profile 防止每次重启都要执行source /etc/profile

#安装nginx
1 安装各种依赖的库

yum install gcc-c++

yum install pcre pcre-devel

yum install zlib zlib-devel

yum install openssl openssl-devel

2 下载nginx,我下载的是最新的稳定版

3 解压

tar -zxvf nginx-1.16.1.tar.gz

4 进入解压后的文件夹

cd nginx-1.16.1

5 执行三个命令进行安装

./configure

make

make install

6 安装完毕后找到默认的安装路径

whereis nginx

7 进入上面查出来的安装路径

cd /usr/local/nginx

8 进入sbin目录

cd sbin

9 启动nginx

./nginx

10 查看nginx进程

ps -ef | grep nginx

一般来说如果防火墙开启了80端口直接在浏览器输入linux的ip地址就能访问到nginx的页面了。
开启防火墙80端口的方式

firewall-cmd --zone=public --add-port=80/tcp --permanent //具体意思见上面tomcat那节的解释
systemctl restart firewalld.service //重启防火墙
firewall-cmd --reload //重新载入防火墙配置

#安装mysql
1 centos7默认使用的是mariadb为什么看这里https://blog.csdn.net/H12KJGJ/article/details/79616761
但是我要讲的是安装mysql，所以首先卸载mariadb
2 查看mariadb是否存在

rpm -qa | grep mariadb

3 删除mariadb

rpm -e mariadb-libs --nodeps

4 解压遇到问题

//这样解压失败
tar -zxvf mysql-8.0.17-1.el7.x86_64.rpm-bundle.tar
提示：
gzip: stdin: not in gzip format
tar: Child returned status 1
tar: Error is not recoverable: exiting now

然后解决办法就是去掉z如下：

tar -xvf mysql-8.0.17-1.el7.x86_64.rpm-bundle.tar

解压出好几个rpm文件
5 执行安装命令，安装这几个rpm文件

yum localinstall -y  mysql-community-{server,client,common,embedded,libs}-*

6 安装完后再次查询是否有了mysql

rpm -qa | grep mysql

可以看到这几个都安装了
mysql-community-common-8.0.17-1.el7.x86_64
mysql-community-client-8.0.17-1.el7.x86_64
mysql-community-server-8.0.17-1.el7.x86_64
mysql-community-embedded-compat-8.0.17-1.el7.x86_64
mysql-community-libs-8.0.17-1.el7.x86_64
mysql-community-libs-compat-8.0.17-1.el7.x86_64

7 下面的命令把/var/lib/mysql的拥有者改为当前用户：root

chown -R root:root /var/lib/mysql

然后重启mysql

systemctl restart mysqld.service

注意：：：mysql的重启、开始、停止命令分别是
systemctl restart mysqld.service
systemctl start mysqld.service
systemctl stop mysqld.service

8 生成临时密码

grep 'temporary password' /var/log/mysqld.log

返回2019-09-20T00:33:37.151972Z 5 [Note] [MY-010454] [Server] A temporary password is generated for root@localhost: XXX我是临时密码XXX
9 登录mysql 执行下面命令然后提示输入上一步的密码

mysql -uroot -p

10 登录成功第一件事修改密码

ALTER USER "root"@"localhost" IDENTIFIED BY "888888";

如果密码过于简单会提示
ERROR 1819 (HY000): Your password does not satisfy the current policy requirements
但是复杂的密码我是真的记不住啊，所以必须改。
我的解决方法是：mysql下执行
设置验证密码验证策略为0

set global validate_password.policy=0;

设置密码验证长度为4

set global validate_password.length=4;

设置简单密码：1234

ALTER USER "root"@"localhost" IDENTIFIED BY "1234";
flush privileges; //刷新mysql权限

成功！
11 远程登录
可能是你的账号不允许从远程登录，只能在localhost。这个时候只要在localhost的那台电脑，登入mysql后更改‘mysql’数据库里的‘user’表里的‘host’项，从‘localhost’改成‘%’就都可以访问了

use mysql;
update user set host = '%' where user = 'root';
flush privileges; //刷新mysql权限

12 首先退出mysql，使用命令

exit;

开启防火墙的3306端口

firewall-cmd --zone=public --add-port=3306/tcp --permanent   //开启3306端口
systemctl restart firewalld.service //重启防火墙

13 设置mysql 开机启动

systemctl enable mysqld
systemctl daemon-reload

14 设置永久表的默认存储引擎和设置字符集
找到mysql的配置文件位置

find / -name my.cnf

打开配置文件

vi /etc/my.cnf

按 i 进入编辑模式在最后加入下面两句话

default-storage-engine=INNODB
character-set-server=utf8

按Esc键 输入 :wq 保存退出
重启mysql

systemctl restart mysqld.service

#安装Git
1.安装依赖

yum -y install zlib-devel openssl-devel cpio expat-devel gettext-devel curl-devel perl-ExtUtils-CBuilder perl-ExtUtils-MakeMaker

2.删除已有git

yum remove git

3.下载git安装包

wget https://mirrors.edge.kernel.org/pub/software/scm/git/git-2.23.0.tar.gz

4.解压

tar -zxvf git-2.23.0.tar.gz

5.进入解压后的目录

cd git-2.23.0

6.配置git的安装目录

make prefix=/usr/local/git all

7.安装

make prefix=/usr/local/git install

8.添加环境变量

echo "export PATH=$PATH:/usr/local/git/bin" >> /etc/bashrc

9.使环境变量生效

source /etc/bashrc

10.查看git版本

git version

11.版本回退

先查看提交的版本
git log
再选择回退的版本
git reset --hard HEAD 版本号
或者这样回退上一个版本，^几个这个回退几个版本
git reset --hard HEAD^

#安装redis
1.redis官网查看最新版本

https://redis.io/download

2.下载redis

wget http://download.redis.io/releases/redis-5.0.5.tar.gz

3.解压

tar xzf redis-5.0.5.tar.gz

4.解压后，在/usr/local目录下创建redis目录。

mkdir /usr/local/redis

5.将redis-5.0.5移动到/usr/local/redis目录下

mv redis-5.0.5 /usr/local/redis

6.进入/usr/local/redis目录

cd /usr/local/redis

7.创建两个文件夹

mkdir etc
mkdir bin

8.进入解压后的目录

cd redis-5.0.5

9.编译

make

编译完成之后，可以看到解压文件redis-5.0.5 中会有对应的src文件夹

10.编译成功后，进入src文件夹，执行make install进行Redis安装。

cd src
make install

11.为了方便管理，将Redis文件中的conf配置文件和常用命令移动到统一文件中
回到redis-5.0.5目录，找到redis.conf，将其复制移动到 /usr/local/redis/ect 下

cd ../

mv redis.conf /usr/local/redis/etc/

12.进入src目录，移动mkreleasehdr.sh redis-benchmark redis-check-aof redis-check-rdb redis-cli redis-server到/usr/local/redis/bin/

cd src

mv mkreleasehdr.sh redis-benchmark redis-check-aof redis-check-rdb redis-cli redis-server /usr/local/redis/bin/

13.设置后台启动redis

cd /usr/local/redis/etc
vim redis.conf

14修改如下几条

daemonize yes #表名需要在后台运行
protected-mode no
bind 0.0.0.0 #开启所有访问，不只是本地
requirepass 123456 #设置密码

其中requirepass不好找直接/requirepass搜索

15.启动redis服务，并指定启动服务配置文件

cd /usr/local/redis/bin
./redis-server /usr/local/redis/etc/redis.conf

16.最坑的还是阿里云，怎么也连接不上，访问不到阿里云上安装启动好的redis，防火墙端口什么的都弄好了，就是访问不了。原来要在策略组加上6379。。。。。。。。

centos7 防火墙添加开放端口：【单个】
firewall-cmd --zone=public --add-port=6379/tcp --permanent
【重新载入防火墙配置】
firewall-cmd --reload
